🌐Browser
WebSocket + REST
📱Mobile
WebSocket + REST
JWT Auth  |  WS + HTTPS
🔀  ALB Ingress Controller
Routes HTTP → REST  |  WS → Django Channels
🐍 Django App Pods (2–3)
  • ASGI via Daphne / Uvicorn
  • REST API (DRF) — Org, Users, Chat
  • Django Channels — WebSocket consumers
  • JWT auth (djangorestframework-simplejwt)
  • Django Signals → Celery tasks
stateless horizontally scalable
🔴 Redis
  • Streams — chat message log per chatID
  • Channel Layer — WS fan-out across pods
  • Celery Broker — async task queue
stream key: chat:{chatID}
  • Consumer groups for replay/history
  • Single Redis instance serves all 3 roles
🐘 PostgreSQL
  • Organisation — name, status, owner
  • User — org FK, role
  • AccessGroup — org FK, members
  • Chat — UUID, org_a, org_b
  • ChatParticipant — chat FK, user FK
persistent store
⚙️ Celery Worker Pods
  • Email on org approval
  • Notifications on chat creation
  • Default access group assignment
  • Any async side-effect from signals
Redis broker retryable
🔐 Auth & Tenant Flow
  • Org registers → status: pending
  • App admin approves → status: approved
  • Org owner creates users (no admin needed)
  • Owner assigns users → AccessGroup
  • JWT issued per user, scoped to org
  • Cross-tenant chat: org_a + org_b users
🧪 Testing (pytest)
  • pytest-django — models, views, signals
  • pytest-asyncio — async consumers
  • channels.testing — WebSocket client
  • factory_boy — test data
  • Separate test DB (auto by pytest-django)
  • Real Redis for integration tests
